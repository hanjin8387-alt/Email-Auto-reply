## Current State Assessment

MailTriageAssistant는 WPF 기반 2컬럼(2:3) 레이아웃의 이메일 분류 대시보드이다.

**구조 개요:**
```
┌─────────────────────────────────────────────────────────────────────┐
│ [Run Triage Now] [Copy Digest & Open Teams]  Teams 이메일: [______] │ ← 상단 툴바 (DockPanel)
│                                               [ProgressBar ====]   │
├───────────────────────┬─────────────────────────────────────────────┤
│ Inbox Priority (2*)   │  제목 (18pt)                           (3*)│
│ ┌───────────────────┐ │  발신자 • 시간 • ActionHint                │
│ │ 85  Action        │ │                                             │
│ │ 홍길동            │ │  Redacted Body Snippet                      │
│ │ 긴급 보고서 요청  │ │  ┌─────────────────────────────────────────┐│
│ │ 2026-02-15 09:30  │ │  │ (마스킹된 본문 요약)                    ││
│ ├───────────────────┤ │  │                                         ││
│ │ 42  FYI           │ │  └─────────────────────────────────────────┘│
│ │ 김철수            │ │                                             │
│ │ 주간 리포트       │ │  Reply Template   [Copy For Copilot]        │
│ │ 2026-02-14 17:00  │ │  [ComboBox ▼]     [Reply With Template]     │
│ └───────────────────┘ │                   [Copy Digest & Teams]     │
├───────────────────────┴─────────────────────────────────────────────┤
│ 상태: 메일 12개 로드 완료                                            │
└─────────────────────────────────────────────────────────────────────┘
```

**현재 상태의 주요 특징:**
- 하드코딩된 hex 색상 다수 사용 (`#DDD`, `#EEE`, `#FAFAFA`, `#666`, `#333`, `#777` 등)
- `AutomationProperties` 미적용 → 스크린 리더 접근 불가
- `ToolTip` 미적용 → 버튼/요소 목적 파악 어려움
- 빈 리스트 상태 안내 UI 없음
- 에러 상태 시 `MessageBox`만 사용, 인라인 안내 없음
- 코드비하인드(`MainWindow.xaml.cs`)는 31줄로 최소화되어 있으나 서비스 생성을 직접 수행 (DI 미적용)
- `ScoreToColorConverter`의 색상 팔레트가 색각 이상자를 고려하지 않음

### 🔴 Critical

| # | Area | Issue | Impact | Recommendation |
|---|---|---|---|---|
| C-1 | 접근성 | 전체 XAML에 `AutomationProperties.Name` 미적용 — 버튼 4개, TextBox 1개, ListBox 1개, ComboBox 1개 모두 스크린 리더 미지원 | 시각 장애 사용자가 앱을 전혀 사용할 수 없음 | 모든 인터랙티브 요소에 `AutomationProperties.Name` 추가. 예: `<Button AutomationProperties.Name="이메일 분류 실행">` |

| C-4 | 사용성 | 이메일 선택 시 본문 로딩 중 피드백 없음 — `LoadSelectedEmailBodyAsync`는 StatusMessage만 변경, `IsLoading` 미설정 → ProgressBar 미표시, 바인딩된 Body 영역에 변화 없음 | 본문 로딩에 수 초 소요 시 사용자가 앱이 멈춘 것으로 인지 | `LoadSelectedEmailBodyAsync` 시작 시 `IsLoading = true`, 완료 시 `IsLoading = false` 설정. 또는 상세 패널 내부에 별도 로딩 인디케이터 추가 |
| C-5 | 접근성 | 키보드 네비게이션에 Tab 순서(TabIndex) 미지정 — WPF 기본 순서에 의존, 논리적 흐름과 불일치 가능 | 키보드 사용자가 비효율적인 순서로 탐색하게 됨 | 주요 인터랙티브 요소에 `TabIndex` 지정: 버튼(1,2) → 이메일(3) → TeamsEmail(4) → 템플릿(5) → 하단 버튼(6,7,8) |

| M-7 | 사용성 | 버튼의 `CanExecute` 상태가 시각적으로 구분되지 않음 — WPF 기본 비활성화 스타일은 미미한 회색 처리만 수행 | 사용자가 왜 버튼을 누를 수 없는지 인지 불가 | 비활성화 시 `ToolTip`에 사유 표시 (예: "먼저 이메일을 선택해주세요"), 비활성 버튼에 커스텀 스타일 적용 |
| M-8 | 시각 디자인 | 카테고리(`Action`, `VIP`, `Meeting` 등)가 텍스트로만 표시 — 12pt 작은 회색(`#666`) 텍스트로 7가지 카테고리 구분 | 카테고리 시각적 구분이 어려워 빠른 스캔에 부적합 | 카테고리별 색상 뱃지 또는 아이콘 추가. 예: `Action` → 🔧, `VIP` → ⭐, `Meeting` → 📅 등 |
